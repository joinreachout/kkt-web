# Optimus Strategy & Scenarios — operating modes

`/solutions/optimus#strategy` (callout on the same page)

## What this is

The same Optimus forecast engine runs in five operating postures, or
"modes". Each mode produces a different recommendation set against
the same operational reality — visible side-by-side in the operator
surface (Strategy & Scenarios screen). The user picks the mode that
matches the moment.

## The five modes

1. **Standard Safety** — default. Conservative buffers, ~45-day
   horizon, low surprise risk. Most networks run on this most of
   the time.

2. **Cost Optimization** — push for the cheapest landed price across
   the network. Tighter buffers, more sensitivity to supplier price
   shifts, more frequent re-balancing.

3. **Cash Conservation** — minimise capital lock. Lower inventory
   days on hand, smaller PO sizes, accept slightly tighter operating
   margin in exchange for free working capital.

4. **Balanced** — median posture between cost, cash, and risk. Useful
   default when conditions are stable and no specific pressure
   dominates.

5. **Crisis** — hold-the-line mode. Survive the immediate window;
   defer everything that can be deferred. Activated when supply,
   logistics, or cash conditions go sharply against the network.

## Why this matters

Conventional procurement systems give one recommendation. Optimus
gives five — one per posture — so the head of supply or the CFO can
see the trade-off explicitly and choose. No black-box "the model
says". The mode comparison is auditable: same inputs, different
recommendations, visible delta in projected cost / capital lock /
fill level / inventory days.

This is the CFO-grade view of the system. It also closes a common
gap: many procurement tools optimise for cost only, leaving the cash
posture and risk posture invisible. Optimus surfaces all three as
explicit choices.
